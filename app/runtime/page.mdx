# Runtime

Runtime is a Rust execution engine that runs task packages, exposes HTTP and gRPC APIs, and integrates with Plane for package fetch, execution status, and credits.

## Responsibilities

- Package lifecycle and local cache management
- Sandboxed JavaScript execution
- Execution queueing and concurrency control
- Output adapter dispatch
- Runtime metrics and health probes
- Plane service integration with HMAC authentication (Runtime -> Plane)

## CLI and Server Modes

- CLI operations (`run`, `list`, `validate`, `lint`, `test`, `profile`, `deps`, etc.)
- HTTP server (`http` command)
- gRPC server (`grpc` command)
- hybrid mode (`serve` command)

## Runtime API Groups

- health and metrics
- package management and execution
- execution query/cancel
- resource tier operations
- adapter output endpoints

## Deployment Boundary

- Runtime HTTP API does not enforce request-level auth by default.
- Production should keep Runtime in private/internal network zones.
- Plane should be used as the public control-plane/API boundary.

## Quality Gate

Run before release-impacting Runtime changes:

```bash
cargo fmt --check
cargo check --locked --all-targets
cargo check --locked --all-targets --no-default-features
cargo test --locked --all-targets
```
