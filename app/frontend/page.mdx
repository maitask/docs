# Plane Frontend

Plane Frontend is a React + Vite console application that manages user and operator interactions with Plane and Runtime.

## Route Domains

Public routes:

- `/`
- `/products`
- `/pricing`
- `/login`
- `/register`

Console routes:

- `/dashboard`
- `/packages`
- `/executions` and `/executions/:id`
- `/workflows` and `/workflows/:id`
- `/schedules`, `/schedules/new`, `/schedules/:id`, `/schedules/:id/history`
- `/playground`
- `/analytics`, `/usage`, `/uptime` (admin domains)
- `/credits`, `/billing`
- `/adapters`
- `/notifications`
- `/organizations`, `/roles`
- `/settings`

## Interaction Model

- Uses token-based auth persisted in browser storage and synced cookie for WebSocket auth.
- Uses unified API wrappers for Plane and Runtime with timeout, cache deduplication, and normalized error handling.
- Supports real-time execution updates through Plane WebSocket endpoints.
- Workflow editor includes trigger, condition, transform, adapter, notification, and completion node types.

## Billing And Credits UX Contract

- Billing and Credits pages dynamically load payment capabilities from `GET /payments/config`.
- Provider lists are runtime-driven (`stripe`, `paypal`, optional `alipay`, `wechat_pay`) instead of hardcoded.
- Free mode fallback paths are supported:
  - `mock` provider when `payment_enabled=false` and `mock_enabled=true`
  - `manual` provider when `payment_enabled=false` and mock is disabled
- Credits top-up handles both backend response modes:
  - checkout redirect when `payment_url` is returned
  - immediate success when `payment_url` is absent (direct top-up)
- Config-load failure handling is fail-closed for payment pages to avoid silently switching production behavior.

## Admin and Access Control

- `PrivateRoute` guards all console pages.
- Admin-only pages are enforced in both UI navigation and backend authorization.
- Permission-sensitive actions (package admin, roles, system metrics) require elevated privileges.
